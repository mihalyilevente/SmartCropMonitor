# Code of Conduct

## 1. Purpose

This project is focused on:

* scientific correctness
* engineering quality
* reproducible results

All interactions are expected to support these goals.

---

## 2. Expected Behavior

Participants must:

* Communicate clearly and technically
* Justify claims with data, metrics, or references
* Keep discussions focused on the problem
* Accept critique of ideas, models, and implementations
* Provide reproducible evidence when proposing changes

---

## 3. Unacceptable Behavior

The following is not allowed:

* Personal attacks or insults
* Non-technical arguments in technical discussions
* Repeated claims without evidence
* Misrepresentation of results or metrics
* Introducing unverifiable “black box” changes
* Ignoring project standards (validation, reproducibility)

---

## 4. Technical Disagreements

Disagreements are resolved using:

1. Empirical evidence (metrics, benchmarks)
2. Reproducibility
3. Code-level inspection
4. Simplicity and clarity of implementation

If unresolved:

* Open a structured discussion (issue or PR)
* Provide experiments or logs

---

## 5. Contribution Standards

All contributions must:

* Follow project architecture
* Be testable and reproducible
* Include explanation of decisions
* Avoid unnecessary complexity

ML-specific:

* Report metrics (Dice, IoU, etc.)
* Provide dataset and preprocessing details

---

## 6. Review Process

Maintainers may:

* Request changes
* Reject contributions lacking justification
* Prioritize stability over experimental features

Reviews focus on:

* correctness
* clarity
* reproducibility
* impact on pipeline

---

## 7. Enforcement

Violations may result in:

* Request for correction
* Rejection of contribution
* Removal from collaboration (in repeated cases)

---

## 8. Scope

This Code of Conduct applies to:

* GitHub issues
* Pull requests
* Code reviews
* Project discussions

---

## 9. Practical Principle

> Strong claims require strong evidence.

---

## 10. Summary

* Be precise
* Be reproducible
* Be technically grounded
* Keep discussions signal > noise

---
